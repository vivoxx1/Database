PARTITIONED INDEX NOTES:
  • Owner - Owner of the partitioned table
  • Index Name - Name of the partitioned index
  • Partitioning Type - Partitioning algorithm
  • Partition Count - Number of partitions in this table
  • Partitioning Key Count - Number of columns in the partitioning key
  • Locality - Whether this partitioned index is LOCAL or GLOBAL
  • Alignment - Whether this partitioned index is PREFIXED or NON-PREFIXED
  • Tablespace - Default TABLESPACE, used for ADD partition
  • Pctfree - Default PCTFREE, used for ADD partition
  • Initrans - Default INITRANS, used for ADD partition
  • Maxtrans - Default MAXTRANS, used for ADD partition
  • Initial Extent - Default INITIAL, used for ADD partition, `DEFAULT' if attribute was not specified
  • Next Extent - Default NEXT, used for ADD partition, `DEFAULT' if attribute was not specified
  • Min Extents - Default MINEXTENTS, used for ADD partition, `DEFAULT' if attribute was not specified
  • Max Extents - Default MAXEXTENTS, used for ADD partition, `DEFAULT' if attribute was not specified
  • Pct Increase - Default PCTINCREASE, used for ADD partition, `DEFAULT' if attribute was not specified
  • Freelists - Default FREELISTS, used for ADD partition
  • Freelist Groups - Default FREELIST GROUPS, used for ADD partition
  • Logging - Default LOGGING attribute, used for ADD partition
  • Buffer Pool - Default buffer pool for the given object, used for ADD partition
